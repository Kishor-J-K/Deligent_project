# Mini E-commerce Data Project

This project generates synthetic e-commerce data, loads it into a SQLite database, and runs a multi-table SQL report.

Structure

/
- data/              # Target folder where CSVs are written (generated by script)
- src/
    - generate_data.py   # Generates CSVs using Faker
    - load_to_sqlite.py  # Creates ecom.db and loads CSVs into SQLite
    - run_query.py       # Executes the SQL in query.sql and prints results
- query.sql          # SQL report joining the tables
- README.md
- .gitignore
- requirements.txt

Quickstart

1) Create and activate a Python virtual environment (Windows PowerShell):

```powershell
python -m venv .venv; .\.venv\Scripts\Activate.ps1
```

2) Install dependencies:

```powershell
python -m pip install --upgrade pip; pip install -r requirements.txt
```

3) Generate the CSV data:

```powershell
python src\generate_data.py
```

This writes CSV files to the `data/` folder: `customers.csv`, `products.csv`, `orders.csv`, `order_items.csv`, `payments.csv`.

4) Load into SQLite:

```powershell
python src\load_to_sqlite.py
```

This creates `ecom.db` in the project root and loads tables. It also validates row counts.

5) Run the report

```powershell
python src\run_query.py
```

This reads `query.sql`, runs it against `ecom.db`, and prints the top 20 joined rows.

Pushing to GitHub

If you want to push this project to your GitHub repository, run these commands (replace the URL):

```powershell
git init
git add .
git commit -m "Add e-commerce data generator and SQLite loader"
# replace <REMOTE_URL> with your GitHub repo URL
git remote add origin <REMOTE_URL>
git branch -M main
git push -u origin main
```

Notes

- `ecom.db` is added to `.gitignore` to avoid committing a binary DB file. If you want to commit generated CSV files, you may remove them from `.gitignore`.
- The generator uses Faker and a deterministic seed for reproducible data.
